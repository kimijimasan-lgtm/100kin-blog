# 開発者向け：次作アプリ登録ガイド

対象: しろちゃん本人（または信頼できる共同開発者）  
更新日: 2026-06-14

---

## 新しいアプリを登録する手順

### STEP 1: 素材を準備する

以下のファイルを用意してください。

| 素材 | サイズ | 説明 |
|------|--------|------|
| サムネイル画像 | 600×400px 推奨 | ホーム画面に表示される代表画像 |
| スクリーンショット | 最大5枚 | 詳細ページに表示 |
| アプリアイコン | 512×512px | ホーム画面の角丸アイコン |
| 紹介文（日本語） | 200〜400文字 | 詳細ページの説明欄 |
| 特徴（箇条書き） | 3〜5項目 | 「このアプリでできること」 |
| 価格 | 数字（円） | 0円＝無料、それ以外＝有料 |
| デモURL | URLまたはnull | ゲストが試せるURL（任意） |

---

### STEP 2: Firestoreにデータを登録する

Firebase コンソール → Firestore → `apps` コレクションに以下の形式で追加：

```json
{
  "id": "app_003",
  "name": "アプリの名前",
  "catchphrase": "一行キャッチコピー",
  "description": "詳しい説明文をここに書く。",
  "price": 300,
  "features": [
    "特徴その1",
    "特徴その2",
    "特徴その3"
  ],
  "thumbnail": "https://storage.googleapis.com/...サムネイルURL",
  "screenshots": [
    "https://...スクリーンショット1",
    "https://...スクリーンショット2"
  ],
  "rating": 0,
  "reviewCount": 0,
  "published": false,
  "createdAt": "2026-06-14"
}
```

※ `published: false` のままでは公開されません。確認後に `true` に変更してください。

---

### STEP 3: 画像をStorage にアップロードする

1. Firebase コンソール → Storage → `apps/app_003/` フォルダを作成
2. サムネイル・スクリーンショットをアップロード
3. 「公開URL」をコピーしてFirestoreのURLに貼り付ける

---

### STEP 4: 動作確認

1. ローカル環境でホーム画面を開き、新しいサムネイルが表示されているか確認
2. 詳細ページを開き、説明・価格・スクリーンショットが正しいか確認
3. ゲストログインで制限が効いているか確認
4. 問題なければ `published: true` に変更

---

### STEP 5: Stripe 決済リンクを設定する（有料アプリのみ）

1. Stripe ダッシュボード → 「支払いリンク」を作成
2. 商品名・価格を設定
3. 購入完了後のリダイレクト先を `https://100kin-apps.com/thanks?app=app_003` に設定
4. 生成されたURLをFirestoreの `stripeLink` フィールドに追加

---

## アプリID命名規則

| ID | アプリ名 |
|----|---------|
| app_001 | PCスマホ連動メモ |
| app_002 | 糖尿人のカウンター |
| app_003 | （次作） |

## チェックリスト

- [ ] サムネイル画像（600×400px）準備完了
- [ ] スクリーンショット（最低1枚）準備完了
- [ ] 説明文・特徴・価格 決定済み
- [ ] Firestore登録完了（published: false）
- [ ] Storage画像アップロード完了
- [ ] 動作確認完了
- [ ] Stripe支払いリンク設定完了（有料の場合）
- [ ] published: true に変更して公開
