## 3. 技術スタック

| 要素 | 選択肢 | 理由 |
|------|--------|------|
| ホスティング | Firebase Hosting | 既存アカウントあり・無料枠 |
| 認証 | Firebase Auth | Google/メール認証対応 |
| DB | Firestore | リアルタイム・スケーラブル |
| 決済 | Stripe | iOS購入はStripe Payment Links経由 |
| メール送信 | Firebase + SendGrid または Resend | 一斉送信に対応 |
| メールアドレス | 専用Gmail（例: apps100kin@gmail.com） | ブログ専用 |

---

## 4. ログイン条件分岐

```
アプリ詳細 → ボタン押下
     ├── ゲストでログイン
     │     → Firebase匿名認証
     │     → パネル3枚・カード10枚に制限
     │     → 上限到達で「購入してください」バナー表示
     └── 購入してログイン
           → Stripeで決済
           → Firebase Auth（メール/Google）
           → Firestoreにpurchased=trueを記録
           → フル機能解放
```

## 5. iPhone対応方針

- PWA対応（manifest.json + service worker）
- レスポンシブCSS（min-width: 320px〜）
- Stripeの支払いリンクはSafariで開く形式（App Store課金を回避）
- ホーム画面に追加可能なアイコン設定

## 6. 大量アクセス対策（YouTuber紹介時）

### 問題: 突発的トラフィック集中
YouTuberに紹介してもらうと、数時間で数千〜数万アクセスが来る可能性がある。

### 対策メニュー

| 対策 | 具体的な手段 | コスト |
|------|------------|--------|
| ① CDN配信 | Firebase Hosting は自動CDN | 無料 |
| ② ウェイティングリスト | アクセス集中時に「今すぐ試す」ボタン → メアド入力 → 順番待ちメール送信 | 低 |
| ③ Cloudflare導入 | DDoS防御 + キャッシュ | 無料〜 |
| ④ 事前ウォームアップ | 紹介動画の公開日前にサーバースケール確認 | 運用対応 |
| ⑤ ゲスト制限を活用 | ゲストは匿名なのでDB書き込み少 → 負荷低減 | 設計による |

**推奨手順（紹介前にやること）:**
1. Firebase Hosting + Cloudflare の組み合わせ設定
2. ウェイティングリストフォームをトップページに設置
3. Firestore のセキュリティルール確認
4. Firebase の予算アラートを設定（$10など）
